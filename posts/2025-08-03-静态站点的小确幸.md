# 静态站点的小确幸

托管在 GitHub Pages 上，静态又自由，  
加载速度像呼吸一样轻，  
每一次刷新都是小确幸的开始。

鸽鸽～把你的静态 Markdown 博客发布到 GitLab，其实超级简单，比喻成“把小纸鸢放到网上，让大家都能看到”💨✨

我给你整理一个完整流程：

---

## 1️⃣ 注册并创建仓库

1. 登陆 [GitLab](https://gitlab.com/)
2. 点击 **“New Project / 新建项目”** → **Create blank project / 创建空项目**
3. 给仓库起个名字，比如 `gogos-blog`，**Public / 公开**（这样别人也能访问）
4. 创建仓库

---

## 2️⃣ 本地项目准备

你的博客目录结构可能像这样：

```
gogos-blog/
│
├─ index.html
├─ css/
│   └─ style.css
├─ js/
│   └─ app.js
└─ posts/
    ├─ plot/
        ├─ 2025-10-01-凌晨唱给月亮的歌-诗_爱情.md
        └─ ...
```

---

## 3️⃣ 初始化 Git 仓库

在本地项目根目录打开终端：

```bash
git init
git add .
git commit -m "初次提交博客"
```

---

## 4️⃣ 关联远程仓库

假设 GitLab 仓库地址是：

```
https://gitlab.com/你的用户名/gogos-blog.git
```

命令：

```bash
git remote add origin https://gitlab.com/你的用户名/gogos-blog.git
git branch -M main
git push -u origin main
```

> 如果是第一次推送，会要求输入 GitLab 的用户名和 Access Token（或者设置 SSH key 更方便）。

---

## 5️⃣ 启用 GitLab Pages（静态网站托管）

1. 在仓库根目录创建 **`.gitlab-ci.yml`** 文件（GitLab CI/CD 配置）：

```yaml
image: alpine:latest

pages:
  stage: deploy
  script:
    - mkdir .public
    - cp -r * .public
    - mv .public public
  artifacts:
    paths:
      - public
  only:
    - main
```

> 这里 `.public` 是临时文件夹，把所有内容拷贝到 `public`，GitLab Pages 默认部署 `public` 目录。

2. 提交 `.gitlab-ci.yml`：

```bash
git add .gitlab-ci.yml
git commit -m "添加 GitLab Pages CI/CD 配置"
git push
```

---

## 6️⃣ 等待部署

* GitLab 会自动触发 **CI/CD**，构建成功后，你的博客就会被部署。
* 页面 URL 通常是：

```
https://你的用户名.gitlab.io/gogos-blog
```

* 你可以访问这个地址查看博客效果。

---

## 7️⃣ 小贴士

* Markdown 文件更新后，**重新 `git add/commit/push`** 就会自动部署。
* 如果你想用二级域名或者自定义域名，也可以在 GitLab Pages 配置里加。
* CSS、JS、Markdown 都是静态文件，GitLab Pages 非常适合这种纯静态博客。

---

鸽鸽～你现在这个报错其实不是 Git 的问题，而是 **网络连接被重置**，Git 在访问 GitHub 时无法建立 HTTPS 连接。通常原因和解决办法如下：

---

## 1️⃣ 检查网络

* 确认能正常访问 [https://github.com](https://github.com)
* 如果你在国内，GitHub 可能会被墙或者网络不稳定
* 可以尝试：

    * 切换到稳定的网络（家庭、公司、VPN）
    * 使用移动热点测试

---

## 2️⃣ 代理设置（可选）

如果必须通过代理访问 GitHub，可以在 Git 配置里加：

```bash
git config --global http.proxy http://127.0.0.1:7890
git config --global https.proxy http://127.0.0.1:7890
```

> `127.0.0.1:7890` 改成你的代理端口

取消代理：

```bash
git config --global --unset http.proxy
git config --global --unset https.proxy
```

---

## 3️⃣ 检查防火墙或杀毒软件

* 有时候 Windows 防火墙、杀毒软件或者企业网络会拦截 HTTPS
* 临时关闭防火墙/杀毒测试一下

---

## 4️⃣ 换 SSH 推送

比起 HTTPS，更稳定的方式是 **用 SSH key 连接 GitHub**：

1. 生成 SSH key（如果没有）：

```bash
ssh-keygen -t ed25519 -C "你的邮箱@example.com"
```

2. 添加到 GitHub：[SSH Keys 设置](https://github.com/settings/keys)
3. 修改仓库地址为 SSH：

```bash
git remote set-url origin git@github.com:Extendsss/fang.github.io.git
git push -u origin main
```

> SSH 不依赖 HTTPS，也不用每次输用户名和密码，通常更稳定。
